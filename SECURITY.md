# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Brix, please report it responsibly. **Do not open a public issue.**

Instead, please email the security concern to the contact listed on the [organization profile](https://github.com/The-Brix-Foundation).

Include as much detail as possible:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

## Response Timeline

- **Acknowledgment**: Within 48 hours of your report
- **Assessment**: Within 7 days, we will assess the severity and confirm the vulnerability
- **Fix**: We aim to release a patch within 30 days for confirmed vulnerabilities

## Supported Versions

As Brix is in early development, security patches will be applied to the latest version on the `main` branch only.

| Version | Supported |
|---------|-----------|
| `main` (latest) | ✅ |
| Older commits | ❌ |

## Scope

This policy covers the Brix compiler (`brixc`) and all repositories under [The Brix Foundation](https://github.com/The-Brix-Foundation).

Thank you for helping keep Brix safe.
